# Security Policy

## Reporting a vulnerability

Please report security vulnerabilities **privately**. Do not open a public issue for a security problem.

Use GitHub's [private vulnerability reporting](https://docs.github.com/en/code-security/security-advisories/guidance-on-reporting-and-writing-information-about-vulnerabilities/privately-reporting-a-security-vulnerability) on the affected repository:

1. Go to the repo's **Security** tab.
2. Click **Report a vulnerability**.
3. Fill out the advisory form with as much detail as you can (affected version, reproduction, impact).

If you cannot use that flow, email **etknorr@gmail.com** with the details.

## What to expect

- We aim to acknowledge a report within **3 business days**.
- We will keep you updated as we investigate and work on a fix.
- Once a fix is available, we will coordinate disclosure and credit you if you'd like.

## Supported versions

Unless stated otherwise in a specific repository, only the **latest released version** receives security fixes.
